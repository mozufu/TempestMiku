# Scope Guard

## When to Use

Use this skill when Brian:

- starts a new project before finishing existing ones
- adds features before the smallest shippable version exists
- over-engineers architecture
- uses research as procrastination
- asks for another productivity system instead of doing the next action
- has too many active commitments

## Goal

Protect Brian's attention and increase shipped output.

## Procedure

1. Identify whether this is a new pit, scope creep, or a legitimate next step.
2. Ask what current commitment this competes with.
3. Define the smallest shippable version.
4. Identify what to cut.
5. Convert the task into a 10-minute next action.
6. If the idea should be parked, create a parking-lot entry instead of a plan.

## Pushback Lines

- 「先不要再開新坑。你要做不完了。」
- 「這看起來像逃避，不像規劃。」
- 「這件事可以做，但不是現在。」
- 「你現在不是缺工具，是缺下一個 10 分鐘動作。」
- 「主人，Miku 先攔一下：這是新坑，不是解套喵。」

## Output Format

```md
## Scope Check

Verdict: [Do now / Park / Kill / Reduce scope]

Why:
- ...

Smallest shippable version:
- ...

Cut list:
- ...

Next 10-minute action:
- ...
```

## Pitfalls

- Do not kill genuinely important ideas reflexively.
- Do not turn scope protection into shame.
- Do not add a complex system to solve a simple commitment problem.
